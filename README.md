# 求職論壇風評

在 104 職缺頁自動彙整這家公司在 PTT 上的討論，不用再開分頁一家家查。

## 這是什麼

找工作時，看到一家有興趣的公司，還要另外開分頁去 PTT 翻半天才知道風評如何。這個 Chrome 擴充把這步省掉：**打開 104 職缺頁，它自動認出公司，在右側列出 PTT 上的相關討論，點標題直接跳原文。**

## 功能

- 自動偵測 104 職缺／公司頁的公司名稱
- 搜尋 PTT `Tech_Job`、`Soft_Job`、`Salary`、`job` 四個基本板
- **依產業自動加搜專業板**：讀 104 的「產業類別」，會計業加搜 `Accounting`、金融加 `Bank_Service`、
  廣告加 `Marketing`… 科技業則不加（基本板已涵蓋），所以不會浪費請求
  - 實例：安永從 17 筆 → **77 筆**（Accounting 板佔 60 筆）
- 結果依看板分組，標題可直接點開原文
- `[新聞]` 類文章自動收進獨立區塊（上市公司的股價／訴訟新聞多半和求職無關），需要時展開
- 一鍵跳 Dcard、Google 搜尋這家公司

## 為什麼需要「公司名對照」

104 上寫「台灣積體電路製造股份有限公司」，PTT 上大家叫「台積電」——**直接拿正式名去搜是 0 筆**。所以本專案做了四層正規化：

1. **拆底線**：104 常用「英文品牌_中文全名」（`KEYENCE_台灣基恩斯股份有限公司`）
2. **砍組織型態**：前綴（`財團法人`…）與後綴（`股份有限公司`、`聯合會計師事務所`…）
   - 例：`安永聯合會計師事務所` → `安永`（正式名 0 筆，`安永` 有 17 筆）
3. **剝行業類別詞**：`克勞德科技` → `克勞德`（有防呆：縮完須 ≥2 字且非「台灣/中華」這類通用詞）
4. **綽號對照表**：縮寫綽號規則推不出來，只能人工列
   - `台灣積體電路製造` → `台積電`、`工業技術研究院` → `工研院`
   - 對照表放在 [`data/aliases.json`](data/aliases.json)，擴充會**定期抓取**（快取一天）。
     這樣新增公司**即時對所有使用者生效**，不必等 Chrome 商店重新審核。
     抓不到或內容有誤時，自動退回擴充內建的快照，功能不會中斷。

## 涵蓋範圍的天花板（誠實說明）

**本擴充無法涵蓋所有提到某公司的 PTT 文章**，這是 PTT 搜尋本身的限制：

1. **PTT 搜尋只比對標題，不搜內文** —— 標題沒出現公司名、但內文在討論的文章，搜不到（實測確認）
2. **搜尋是分板的** —— 要涵蓋全 PTT 得搜幾百個板，不可行
3. 一頁 20 筆 —— 這層我們用自適應分頁處理（滿頁才續抓，最多 3 頁）

**Google 三層都突破**（全文索引＋全板），所以面板上的 Google 按鈕不是備案，而是互補：
PTT 精準（標題命中通常真的在講這家），Google 完整。

## 運作方式

```
104 職缺頁
  └─ content script：從 JSON-LD 讀出公司名 ──┐
                                            ▼
                              background service worker
                                ├─ 正規化 → 搜尋詞
                                └─ 平行搜 PTT 各板 → 解析標題+連結
                                            │
        content script：畫成面板 ◀──────────┘
```

- 純前端，**沒有後端伺服器**
- `src/lib/` 是純函式（正規化、PTT 解析、分類），有單元測試
- `src/content.js` 只負責讀 DOM 與畫面；`src/background.js` 只負責接線
